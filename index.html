白嫖校园网

Posted on 2019-03-18
Words count in article: | Reading time ≈

瞎逼逼

学校在上课学期开就对校园网进行了改造，原来宿舍的宽带本来可以一个人办理，用个路由器就可以全寝室一起使用。改造之后，每次上网都需要进行认证，而且无法正常通过路由器分享一个人的宽带套餐，每个设备都要单独认证，一个账号也只能同时两个设备同时在线。即使学校的三大运营商给出了一些优惠的套餐，但始终都不方便（其实是贫穷.jpg）,学校也给了一个月8.5的校园网wifi套餐，但2兆的网速确实不够用，于是就像找到一些办法来提升校园网使用质量（白嫖）。

特别情况

刚在校园网开始使用的时候，就有人找到有办法，让一个宿舍可以共享一个账号，现在貌似也只能电信的宽带可以，具体的做法就是，先直接将校园网的网线插在一台电脑上，进行认证后，获得到电脑的mac地址，并手动修改路由器的mac地址，时路由器和认证的电脑mac地址相同，然后将网线直接插在路由器上，就可以通过修改mac的方式欺骗认证系统，实现网络共享

正文

就在不久前，当我躺在床上看番卡的要死时，突然产生了一个想法，学校虽然无线网最便宜的套餐只有2兆的网速，但是如果只是访问学校内网，其实是不会限速的。而在学校办公室或者实验室之类的地方，还是走的以前的校园网认证，这个认证后的网络也是不用限速的，于是产生了在办公室做一个代理的想法。

Markdown是一种纯文本格式的标记语言。通过简单的标记语法，它可以使普通文本内容具有一定的格式。

相比WYSIWYG编辑器

优点：
1、因为是纯文本，所以只要支持Markdown的地方都能获得一样的编辑效果，可以让作者摆脱排版的困扰，专心写作。
2、操作简单。比如:WYSIWYG编辑时标记个标题，先选中内容，再点击导航栏的标题按钮，选择几级标题。要三个步骤。而Markdown只需要在标题内容前加#即可

缺点：
1、需要记一些语法（当然，是很简单。五分钟学会）。
2、有些平台不支持Markdown编辑模式。

hexo博客的安装

window10下安装

相关步骤
  1. 安装Node.js并配置好Node.js环境，并使用cmd输入命令查看

  1. 安装Git并配置Git环境，直接从官网下载安装，安装好后会又如图应用

  1. Github建立项目，项目必须要遵守格式：账户名.github.io，并勾选Initialize this repository with a README

建好这个项目后，得到一个网址: saltedfishhuang.github.io,可以直接访问，但页面404

  1. 安装Hexo，在合适的位置创建一个存博客的文件夹，因为我已经装好了，所以下面都借用网图，首先在cmd中cd到创建的文件夹下输入：npm install hexo -g

输入hexo -v,检查hexo是否安装成功

输入hexo init,初始化该文件夹

输入npm install,安装所需要的组件

输入hexo g ,本地生成页面

输入hexo s 开启本地服务，访问给出地址，可以本地浏览博客

如果长期无法跳转页面，可能是端口被占用，使用ctrl+c停止服务,输入hexo server -p 端口号来改变端口号

出现下图则成功

  1. 在博客文件夹下使用Git Bash，将Hexo和Github page联系起来

    输入ssh-keygen -t rsa -C “huangjiacheng528@gamil.com”，连续三个回车，生成密钥，最后得到了两个文件：id_rsa和id_rsa.pub（默认存储路径是：C:\Users\Administrator\.ssh）

hexo简单安装

输入eval "$(ssh-agent -s)"，添加密钥到ssh-agent

再输入ssh-add ~/.ssh/id_rsa，添加生成的SSH key到ssh-agent

在Github上，点击头像下的settings，添加ssh，新建一个new ssh key，将id_rsa.pub文件里的内容复制上去

  1. 配置Deployment，在blog文件中中修改repo值

  2. 在生成及部署博客前，安装扩展 npm install hexo-deployer-git --save

​ 使用命令：hexo d -g，生成以及部署博客，访问http://用户名.github.io

工作室因为本身性质的一些原因，管理相对于学校其他工作室也较为严格，暑假7月初就开始了暑期培训，一直持续到8月10号，也快要熬到头了。每天早7晚9的生活也可以暂时停一段时间，迎来我大学生活的第一个暑假。

想了想，反正这是个没人看的博客，自己也一些东西挺好，于是也想把从4月加入工作室，到如今的一些学习经历写一写。

顺着时间线写吧！

4月初，刚进入工作室，课程的安排最开始是C#的和数据库的基础学习，C#是面向对象的语言，此前对面向对象的语言学习还没有怎么接触，学习起来很吃力，进度也很慢，不过所幸只是一些基础的学习，后期页逐渐找到一些感觉。而另外一边的SQL server的学习则要相对简单一些，主要就是学习数据库基础操作的增删查改。

到了六月，工作室的课程仍然在继续还是主要为前端和web，前端从HTML到CSS到javascript，到使用jquery和bootstrap，学习到js后面的内容我渐渐赶到我的学习组建变慢，pentest的学习基础语法一直在持续，但我大部分都是粗略的了解基础语法，根本没仔细学习，而且进度也很慢，同时也开始要求我们开始使用一些常用的工具如fiddler、wireshark、burpshuit、git等等等等，同时六月后半部分也进入了期末复（yu）习的状态，巨佬也明白我们进度太慢，开始停更，让我们专心备考。

算是第一篇-个人搭建此博客的一些心得

Posted on 2018-07-17
Words count in article: | Reading time ≈

其实很早以前就像搭建一个个人站点，但一直没找到机会，就算是以前有过练习，也只是在虚拟机上进行的本地建站而已。

两个月前，我也是一直用别人的SSR，虽然很多人都愿意让我厚颜无耻的蹭一波，但考虑到自己以后学习需要，还是在 DigitalOcean上拥有了自己的第一个水滴（VPS服务器），作为一个贫穷的孩子，当然也用了各种优惠。主要就是GitHub上领取的学生包，中途搞笑的是学校注册的教育邮箱我还无法使用，最后靠直接使用学生证拍照上传拿到了$50的信用额度。在注册VPS服务其实也是一堆问题，我本来想用自己的建行卡直接跨境支付，却告知我，我的银行卡无法支付emmmmm，无法形容当时的心情，最后还是靠小伙伴的PayPal才支付成功，感激不尽。

水滴有了，接下来就是配置SSR，最开始用手动配置的方式，到后面发现代理无法访问，也是很难受了，后面选择使用一键脚本直接搭建的SS服务才成功。

之后这台位于旧金山的服务器也被我这样搁置下来，知道最近才有时间利用起它来搭建一个简单的WordPress个人博客站点，服务器是Ubuntu 16.04,使用的是lamp环境。搭建过程也没什么问题，一路顺利的搭建完成，配置好了数据库，设置成功了主页。

但配置成功后，提醒我WordPress可以更新时，我也就自然而然的想去更新，却提醒我需要为ftp文件传输配置账号。在前面的安装中，我并没有进行ftp的配置，于是自然而然的采取了下列措施apt-get install vsftpd，，安装后发现还要配置用户访问权限，一脸懵逼的百度加谷歌，配置好了ftp文件服务器。发现WordPress还是出现了权限不足，无法在目录下新建文件夹。想来想，就饿得可能还有其他解决方案，在一番搜索下，修改了/var/www/html/目录下的wp-config.php配置文件，增加了一些内容，终于在2018年7月17日晚8点，成功更新。顺势还将我在阿里云上花了一块钱注册的域名加上去，可以顺利访问并添加博客了emmmm。
