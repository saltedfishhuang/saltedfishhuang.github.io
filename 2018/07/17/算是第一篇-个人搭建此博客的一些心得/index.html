算是第一篇-个人搭建此博客的一些心得

其实很早以前就像搭建一个个人站点，但一直没找到机会，就算是以前有过练习，也只是在虚拟机上进行的本地建站而已。

两个月前，我也是一直用别人的SSR，虽然很多人都愿意让我厚颜无耻的蹭一波，但考虑到自己以后学习需要，还是在 DigitalOcean上拥有了自己的第一个水滴（VPS服务器），作为一个贫穷的孩子，当然也用了各种优惠。主要就是GitHub上领取的学生包，中途搞笑的是学校注册的教育邮箱我还无法使用，最后靠直接使用学生证拍照上传拿到了$50的信用额度。在注册VPS服务其实也是一堆问题，我本来想用自己的建行卡直接跨境支付，却告知我，我的银行卡无法支付emmmmm，无法形容当时的心情，最后还是靠小伙伴的PayPal才支付成功，感激不尽。

水滴有了，接下来就是配置SSR，最开始用手动配置的方式，到后面发现代理无法访问，也是很难受了，后面选择使用一键脚本直接搭建的SS服务才成功。

之后这台位于旧金山的服务器也被我这样搁置下来，知道最近才有时间利用起它来搭建一个简单的WordPress个人博客站点，服务器是Ubuntu 16.04,使用的是lamp环境。搭建过程也没什么问题，一路顺利的搭建完成，配置好了数据库，设置成功了主页。

但配置成功后，提醒我WordPress可以更新时，我也就自然而然的想去更新，却提醒我需要为ftp文件传输配置账号。在前面的安装中，我并没有进行ftp的配置，于是自然而然的采取了下列措施apt-get install vsftpd，，安装后发现还要配置用户访问权限，一脸懵逼的百度加谷歌，配置好了ftp文件服务器。发现WordPress还是出现了权限不足，无法在目录下新建文件夹。想来想，就饿得可能还有其他解决方案，在一番搜索下，修改了/var/www/html/目录下的wp-config.php配置文件，增加了一些内容，终于在2018年7月17日晚8点，成功更新。顺势还将我在阿里云上花了一块钱注册的域名加上去，可以顺利访问并添加博客了emmmm。
